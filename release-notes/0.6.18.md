# Added

### #6554 [vividus-plugin-rabbitmq] Add RabbitMQ plugin
New plugin with configurable RabbitMQ broker connections and dedicated messaging steps.

Configuration is defined per broker key (`rabbitmq.<broker-key>.*`):
```properties
rabbitmq.<broker-key>.host=localhost
rabbitmq.<broker-key>.port=5672
rabbitmq.<broker-key>.username=guest
rabbitmq.<broker-key>.password=guest
rabbitmq.<broker-key>.virtual-host=/
```

Core steps added by the plugin:
```gherkin
When I publish message `$message` with routing key `$routingKey` to RabbitMQ broker `$brokerKey`
When I retrieve message from queue `$queue` of RabbitMQ broker `$brokerKey` with `$timeout` timeout and save it to $scopes variable `$variableName`
```

### #6573 [vividus-plugin-rabbitmq] Add ability to use SSL for RabbitMQ connections
New connection property:
```properties
rabbitmq.<broker-key>.use-ssl=false
```
Set to `true` to enable encrypted (TLS) connection to the selected broker.

### #6590 [vividus-plugin-rabbitmq] Add ability to set RabbitMQ message properties
New step:
```gherkin
When I set RabbitMQ message properties:$properties
```
Allows setting AMQP message properties for the next publish operation (the properties are consumed on publish). Supported properties include `contentType`, `contentEncoding`, `correlationId`, `replyTo`, `expiration`, `messageId`, `type`, `userId`, `appId`.

### #6574 [vividus] Add step to wait for file appearance in directory
New step:
```gherkin
When I wait `$timeout` with `$pollingTimeout` polling until file matching `$fileNameRegex` appears in directory `$directoryPath` and save path to $scopes variable `$variableName`
```
Polls a directory until exactly one matching file appears and saves its absolute path to a variable.

# Changed

The step activating an application by bundle identifier now logs the bundle identifier for better traceability.
